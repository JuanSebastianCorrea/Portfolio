The Cook's Nook

An app to help users keep all recipes in one place.

The goal of this project was to create and app that 1) authenticated and authorized users, 2) made requests to an external API, 3) implemented full CRUD, and 4) used Python/Flask for the back end.


Front End

  • HTML
  • CSS
  • JavaScript
  • Bootstrap
  • JQuery
  • Axios

Back End

  • Python
  • Flask
  • SQL Alchemy
  • Bcrypt
  • WTForms

Database

  • PostgreSQL

I decided to create an app that used a mixture of client-side and server-side rendering. All user requests to get new recipes based on search results are rendered on the client-side using AJAX via Axios. The client makes a request to the internal API in Flask, which in turn requests to Edamam's API. This helps maintain the client side decoupled from the logic to hit the external API and allows for error handling on the backend.

Users can also search for saved and personal recipes by typing on the searchbar and have results rendered real-time. Any recipes saved or created by the user are stored in the database in order to optimize this real-time search functionality.

All user pages to view, save, create, update, or delete recipes are rendered server side.
